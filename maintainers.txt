# Beckett maintainers — bundled baseline (intentionally empty).
#
# One Discord user id (snowflake) per line; '#' lines and blanks are ignored.
# A maintainer may have Beckett push, merge, deploy, and restart on request —
# the same authority the owner holds for those specific actions, and nothing more.
#
# Owner-approved additions land in ~/.beckett/maintainers.txt at runtime
# (two-phase: `beckett maintainer grant <id>` files a request, only the OWNER's
# `approve <code>` applies it). Only the owner adds maintainers — maintainers
# cannot add peers or self-elevate.
